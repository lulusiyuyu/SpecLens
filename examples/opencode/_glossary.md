# OpenCode — Glossary

## Terms

| Term | Definition | Also Known As |
|------|-----------|---------------|
| Agent | The orchestrator that runs the LLM ↔ tool-call loop until task completion | LLM Agent |
| Provider | An implementation that communicates with a specific LLM API (Anthropic, OpenAI, etc.) | LLM Provider |
| Tool | A capability the agent can invoke (bash, file edit, file read, search, etc.) | Function |
| Tool Call | An LLM response requesting execution of a specific tool with parameters | Function call |
| Session | A conversation thread with its own message history | Conversation |
| Message | A single chat message (user or assistant) stored in the database | — |
| TUI | Terminal User Interface — the interactive terminal frontend | — |
| Bubble Tea | A Go framework for building terminal applications using the Elm architecture | — |
| Lip Gloss | A Go library for styling terminal output | — |
| sqlc | A tool that generates type-safe Go code from SQL queries | — |
| LSP | Language Server Protocol — a standard for code intelligence tools | — |
| PubSub | Internal publish/subscribe event system for decoupled communication | Event bus |
| Condenser | Not used in OpenCode (OpenHands term) | — |
